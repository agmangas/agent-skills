# Source Prioritization Strategy

Defines the systematic selection process when more than 50 sources are detected. Load this reference after scoring (see `scoring-system.md`) to decide which sources to keep, merge, or exclude.

---

## Detection and Analysis

### Step 1: Count Total Sources

```bash
# Count all compatible files
find . -type f \( -name "*.pdf" -o -name "*.docx" -o -name "*.txt" -o -name "*.md" -o -name "*.csv" \) | wc -l

# List with sizes
find . -type f \( -name "*.pdf" -o -name "*.docx" -o -name "*.txt" -o -name "*.md" -o -name "*.csv" \) -exec du -h {} \; | sort -rh > source_inventory.txt
```

### Step 2: Initial Categorization

```bash
# Group by type/topic (from directory names)
find . -type f -name "*.pdf" | sed 's|/[^/]*$||' | sort | uniq -c

# Create category manifest
mkdir -p .analysis
find . -type f -name "*.pdf" -o -name "*.docx" | while read f; do
  category=$(dirname "$f" | sed 's|^\./||')
  echo "$category|$f" >> .analysis/categorized_sources.txt
done
```

## Decision Matrix

| Total Score | Uniqueness | Action | Reasoning |
|-------------|------------|--------|-----------|
| 35-40 | High (8-10) | **KEEP** | Essential unique content |
| 35-40 | Low (0-4) | **MERGE** | High value but duplicative |
| 25-34 | High (8-10) | **KEEP** | Unique perspective compensates |
| 25-34 | Medium (5-7) | **CONDITIONAL** | Keep if space allows |
| 25-34 | Low (0-4) | **MERGE** | Consolidate with similar |
| 15-24 | Any | **MERGE** | Medium value, merge candidate |
| 0-14 | Any | **EXCLUDE** | Low value, drop or merge |

### Space-Based Adjustments

Tighten or loosen thresholds based on how far over the 50-source limit you are:

- If total sources = 60-80: Keep 35+, MERGE 25-34
- If total sources = 80-150: Keep 30+, MERGE 20-29
- If total sources = 150-300: Keep 35+, MERGE carefully
- If total sources > 300: Keep 37+ only, aggressive merging

## Selection Process Script

```bash
#!/bin/bash
# select_top_50.sh - Systematic source selection
# Reads source_scores.csv (generated by score_sources.sh) and selects
# the top sources to fit within the 50-source limit.

SOURCE_SCORES="source_scores.csv"
TARGET=50

echo "Step 1: Auto-select high scorers (35+)"
awk -F, '$6 >= 35 {print $1}' "$SOURCE_SCORES" | tail -n +2 > .analysis/keep_auto.txt
AUTO_COUNT=$(wc -l < .analysis/keep_auto.txt)
echo "Auto-selected: $AUTO_COUNT sources"

REMAINING=$((TARGET - AUTO_COUNT))
echo "Remaining slots: $REMAINING"

if [ $REMAINING -gt 0 ]; then
  echo "Step 2: Select from 30-34 range by uniqueness"
  awk -F, '$6 >= 30 && $6 < 35 {print $4","$1}' "$SOURCE_SCORES" |
    sort -t, -k1 -rn |
    head -n $REMAINING |
    cut -d, -f2 >> .analysis/keep_auto.txt
fi

# Update remaining
SELECTED=$(wc -l < .analysis/keep_auto.txt)
REMAINING=$((TARGET - SELECTED))

if [ $REMAINING -gt 0 ]; then
  echo "Step 3: Fill remaining with 25-29 range"
  awk -F, '$6 >= 25 && $6 < 30 {print $4","$6","$1}' "$SOURCE_SCORES" |
    sort -t, -k1 -rn -k2 -rn |
    head -n $REMAINING |
    cut -d, -f3 >> .analysis/keep_auto.txt
fi

echo "Final selection: $(wc -l < .analysis/keep_auto.txt) sources"
echo "Sources to keep:"
cat .analysis/keep_auto.txt

# Identify merge candidates (similar scores, same category)
echo -e "\nMerge candidates (similar low-mid scores):"
awk -F, '$6 >= 15 && $6 < 30 {print $1}' "$SOURCE_SCORES" | tail -n +2 > .analysis/merge_candidates.txt
cat .analysis/merge_candidates.txt
```
